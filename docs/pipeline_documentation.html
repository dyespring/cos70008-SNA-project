1. Architecture Overview

The pipeline transforms unstructured text (PDF policy documents + Yelp JSON reviews) into a conceptual network where nodes are key concepts and edges are semantic relationships. It then applies social network analysis (SNA) techniques to extract insights.

3. Pipeline Stages (1–6)

1 Ingestion

Lines 26–59 in pipeline.py

  • Policy: pdf_reader.read_policy_pdf() extracts text per page using pdfplumber, strips headers/footers
  • Yelp: yelp_reader.load_yelp_reviews() reads extracted JSONL, filters by category, samples N reviews
  • Combined: both are loaded into the same document list

2 Preprocessing

Lines 61–64 (summary), cleaning + tokenisation happen during ingestion

  • clean_text() — removes URLs, emails, Unicode artefacts, policy-specific noise
  • SpacyTokeniser — tokenises, lemmatises, tags POS, extracts NER entities & noun chunks
  • Each document gets a source tag ("policy" or "yelp")

3 Extraction

Lines 66–84

  • ConceptExtractor — 3 strategies: Named Entity Recognition, Noun-Phrase chunking, TF-IDF keywords
  • Each concept tracks source_origins → knows if it came from policy, yelp, or both
  • RelationshipExtractor — co-occurrence windows + dependency parsing; classifies as ASSOCIATION / ACTION / CAUSATION

4 Network Construction

Lines 86–96

  • GraphBuilder creates a NetworkX DiGraph from concepts (nodes) and relationships (edges)
  • Edges below --min-weight are pruned
  • Parallel edges of different types are aggregated
  • Exports: network.graphml, edges.csv, nodes.csv

5 Analysis

Lines 98–152

  • Centrality: degree, betweenness, closeness, eigenvector, PageRank
  • Communities: Louvain (preferred) or label propagation
  • Brokerage: nodes bridging communities
  • Cross-source (combined mode): overlap %, bridging concepts, cross-source edges

6 Visualisation

Lines 154–182

  • Static: network graph, top concepts bar chart, community distribution, co-occurrence heatmap, source-coloured network, source overlap chart
  • Interactive: pyvis HTML network with hover tooltips, zoom, drag
  • Combined mode colours nodes by source: Policy Yelp Both

5. Combined Mode & Cross-Source Analysis

When --source combined is used, the pipeline ingests both data sources into a unified corpus. Every concept is tagged with its origin, enabling cross-domain analysis.

Cross-Source Metrics

  • source_distribution — count of concepts per source type
  • source_comparison_summary — policy-only vs yelp-only vs shared, overlap percentage
  • cross_source_edges — every edge connecting nodes from different sources
  • bridging_concepts — top concepts ranked by bridge_score = betweenness × (1 + cross-source neighbour ratio)

8. Usage Examples

# Combined pipeline — 500 Yelp reviews + policy, min-weight 2
PYTHONUNBUFFERED=1 venv/bin/python pipeline.py \
    --source combined --yelp-sample 500 --min-weight 2 \
    --output results/combined_500

# Larger batch — 5000 reviews, stronger edge filter
PYTHONUNBUFFERED=1 venv/bin/python pipeline.py \
    --source combined --yelp-sample 5000 --min-weight 3 \
    --output results/combined_5000

# Policy only
venv/bin/python pipeline.py --source policy --output results/policy_only

# Yelp only, specific category
venv/bin/python pipeline.py --source yelp --yelp-sample 1000 \
    --yelp-category "Food" --output results/yelp_food

# Use only NER + TF-IDF (skip noun phrases)
venv/bin/python pipeline.py --source combined --methods ner tfidf \
    --output results/combined_ner_tfidf
