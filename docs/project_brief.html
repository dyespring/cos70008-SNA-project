COS70008 — Technology Innovation Project · Project Brief

Text-to-Network Engine

From Text to Networks: a Neo4j-only conceptual network engine for social network analysis, with server-side GDS analytics and a pluggable LLM/RAG chatbot grounded in the graph.

Industry partner: SNA Toolbox Python 3 · spaCy Neo4j 5 + GDS (single source of truth) Streamlit dashboard · Chatbot

1Project Summary

The Text-to-Network Engine is a Python pipeline that transforms unstructured text into conceptual networks for social network analysis (SNA). It ingests raw text (policy documents and Yelp reviews), extracts key concepts and their relationships using NLP, writes them straight into Neo4j as (:Concept)-[:RELATED]->(:Concept), and asks the Neo4j Graph Data Science (GDS) plugin to compute centralities, communities, brokerage and paths server-side. A Streamlit dashboard and an LLM/RAG chatbot read the graph back via Cypher to let analysts explore it interactively.

Problem

Analysts have access to large volumes of unstructured text (policy, reviews, narratives) but few tools to convert that text into structured, queryable conceptual networks suitable for SNA.

Solution

An end-to-end, Neo4j-native pipeline that turns documents into a persistent, queryable conceptual graph — and surfaces it through static visuals, an interactive dashboard, and a graph-grounded chatbot, with all SNA computation pushed to the database.

Outcome

A reusable engine plus a reference deployment on two distinct corpora (climate policy and consumer reviews), demonstrating cross-domain applicability and cross-source insight extraction.

3Pipeline at a Glance

1 · Ingest→ 2 · Preprocess→ 3 · Extract→ 4 · Write Neo4j→ 5 · GDS Analyse→ 6 · Visualise→ 7 · Temporal→ 8 · Chatbot

1Data Ingestion

Read policy PDF (pdfplumber) and/or Yelp JSON reviews (pandas).

2Preprocessing

Clean, tokenise, lemmatise and filter text with spaCy (en_core_web_sm).

3Concept & Relationship Extraction

NER + noun-phrase chunking + TF-IDF; co-occurrence windows, dependency parsing and causal pattern matching.

4Write to Neo4j

Neo4jGraphWriter upserts (:Concept)-[:RELATED]-> via batched Cypher UNWIND — no in-memory graph.

5GDS Analysis (server-side)

Neo4j GDS computes PageRank, Louvain, betweenness, closeness, eigenvector, WCC and local clustering, writing them back as node properties.

6Visualisation

Top-N subgraph fetched back via Cypher: static matplotlib plots + interactive pyvis HTML from pipeline.py viz. The Streamlit Network tab renders the same database through vis-network instead.

7Temporal Comparison optional

Per-slice subgraphs tagged with a slice_id property; overlap / jaccard computed via Cypher set operations.

8Conversational Exploration optional

Pluggable LLM/RAG chatbot grounded in the Neo4j graph; semantic search via Neo4j's native node + relationship vector indexes.

4Architecture & Tech Stack

Core libraries

  • spaCy — tokenisation, lemmatisation, NER, dependency parsing
  • scikit-learn — TF-IDF concept ranking
  • Neo4j 5 (APOC + GDS) — single source of truth: stores nodes/edges, vector indexes, and runs SNA algorithms server-side
  • matplotlib + pyvis — static PNG + interactive HTML from pipeline.py viz (offline artefacts under results/)
  • vis-network (via vis_network_html.py) — in-browser graph inside the Streamlit dashboard (same Neo4j data, different renderer than pyvis)
  • Streamlit — analyst-facing dashboard (Network, Centrality, Communities, Brokers, Temporal, Chat, Summary)

Optional / extension layers

  • VADER / TextBlob — sentiment-weighted edges (written back as r.sentiment)
  • sentence-transformers — node + edge embeddings (stored in Neo4j vector indexes)
  • LLM providers — pluggable: OpenAI, DashScope (Qwen), Hugging Face API, local HF models, or echo fallback

5Key Features & Extensions

Neo4j as sole datastore Every node, edge, embedding and metric lives in Neo4j; downstream apps read via Cypher. Always on (required); docker compose up -d
Server-side SNA via GDS PageRank, Louvain, betweenness, WCC etc. run inside Neo4j and write metrics back to nodes. python pipeline.py analyse --source <src>
Sentiment-weighted edges VADER / TextBlob scores written back as r.sentiment via UNWIND. --sentiment
Concept dictionary Analyst-controlled vocabulary boosts/filters key terms. --concept-dictionary default
Temporal comparison Per-slice subgraphs tagged with slice_id; comparison via Cypher set operations. pipeline.py temporal --temporal N · Temporal tab
Streamlit dashboard Reads metadata, centralities, communities and chat from Neo4j directly. python -m streamlit run src/extensions/dashboard.py
LLM / RAG chatbot Conversational Q&A grounded in the Neo4j graph; pluggable LLM providers. python chat.py · pipeline.py chat · Chat tab
Edge-level semantic search Verb-aware retrieval (“who recommended the food?”) via Neo4j relationship vector index. pipeline.py etl --embed + chat --vector-store
Advanced extraction pass Optional deterministic merge of substring-near-duplicate concepts after extraction. --advanced-extraction on etl / shared args
Dashboard LLM narratives Optional short blurbs on metric tabs and polished SNA insight cards (requires LLM env). Set LLM_PROVIDER + keys; use on-demand buttons in Streamlit
Temporal insight cards Comparison / drift style observations over slice_id slices in the Temporal tab. Run pipeline.py temporal (or all --temporal N); inspect Temporal tab

6Deliverables

Software

  • End-to-end CLI pipeline (pipeline.py) with subcommands for ETL, GDS analysis, viz, temporal, chat
  • Standalone Stage-8 chatbot (chat.py) reading from Neo4j
  • Streamlit dashboard with Network, Centrality, Communities, Brokers, Temporal, Chat and Summary tabs

Outputs & artefacts

  • Live conceptual graph in Neo4j (browse via http://localhost:7474)
  • Centralities, communities, brokers and bridging concepts written back as node properties
  • Optional CSV dumps via --export-csv for downstream sharing
  • Static PNG plots and interactive HTML network visualisations from top-N subgraphs

7Quick Start

Install
pip install -r requirements.txt && python -m spacy download en_core_web_sm
Bring up Neo4j
cp .env.example .env && docker compose up -d — set NEO4J_PASSWORD in .env
Policy run
python pipeline.py all --source policy --reset --embed
Yelp run
python pipeline.py all --source yelp --yelp-category Restaurants --yelp-sample 500 --reset
Combined + extras
python pipeline.py all --source combined --reset --sentiment --embed --temporal 3
Dashboard
python -m streamlit run src/extensions/dashboard.py
Chatbot
python chat.py --source policy --vector-store  ·  python pipeline.py chat --source policy
